## What is Anyshift

A Windows-only Python GUI app that converts H-pattern shifter joystick button presses into sequential key presses (or direct memory writes) so vintage racing games work with an H-shifter. Supports optional Arduino serial output for a 7-segment gear display.

## Running and Building

All commands are run from the `src/` directory, since file paths (`Anyshift.ini`, `Resources/`, `Presets/`) are resolved relative to the script's own location.

```bash
cd src
python Main.py          # Run the app in development
```

```bash
cd src
pyinstaller Main.spec   # Build the Windows executable βÜ£ dist/Main.exe
```

**Dependencies** (Windows only): `PySide6`, `pygame`, `keyboard`, `ReadWriteMemory`, `pyserial`

There is no requirements.txt; install manually with pip.

## Architecture

### Module responsibilities

| File | Role |
|------|------|
| `Main.py` | PySide6 `QMainWindow` subclass (`MyWindow`). Entry point. Reads/writes the `options` dict and dispatches to all other modules. |
| `UI.py` | Auto-generated by Qt Designer βÄî **do not edit manually**. Defines `Ui_MainWindow`. |
| `Joystick.py` | Core pygame event loop (`joystickLoop`). Detects button presses per gear, drives key simulation or memory writes, manages Arduino serial. |
| `CtypeKeyPressSimulator.py` | Low-level Windows `SendInput` wrapper using DirectInput scan codes. `sendKeystrokes()` implements H-to-sequential logic (counts up/down from actual gear to target gear). |
| `ReadWriteSaves.py` | `configparser`-based INI read/write. `iniReader()` / `iniWriter()`. Also contains `hexConvert()` / `charConvert()` for char βÜî DirectInput hex string conversion. |
| `Global.py` | Single shared boolean `done` βÄî set to `True` to stop the pygame loop from the GUI thread. |
| `PlaySound.py` | Plays a random gear-grind `.wav` (from `Resources/`) via `pygame.mixer` when clutch is engaged but not pressed. |

### Key design decisions

**DirectInput scan codes, not virtual key codes.** `CtypeKeyPressSimulator.py` uses `KEYEVENTF_SCANCODE` (flag `0x0008`) so keystrokes reach games regardless of focus. Keys are stored in `Anyshift.ini` as hex strings (e.g. `0x1F` for `s`) and displayed as characters in the UI; `hexConvert`/`charConvert` handle conversion.

**Blocking main thread loop.** `joystickLoop()` is called on the GUI thread and blocks it while running. `Global.done = True` (set by the Stop button or the `End` key) breaks the loop and returns control to the GUI.

**Two operating modes**, selected per profile:
- *Key press mode*: Simulates sequential up/down key presses to match the target gear.
- *Memory write mode*: Writes gear integer values directly to a game process address (via `ReadWriteMemory`). Used for DOSBox 0.74 and PCSX2.

**`application_path` pattern.** Both `Main.py` and `PlaySound.py` detect whether they're running as a frozen PyInstaller exe or a plain script to resolve resource paths correctly.

### Configuration / Presets

`Anyshift.ini` (INI format, sections `[SHIFTER]`, `[KEYS]`, `[OPTIONS]`, `[EXTRAS]`) stores the active configuration. A copy lives at the repo root for distribution; the working copy is at `src/Anyshift.ini`.

Preset files (`src/Presets/*.any`) use the identical INI format and can be loaded/saved via the File menu.
